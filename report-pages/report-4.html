Eksperimen: Implementasi ARP Spoofing untuk Man-in-the-Middle Attack pada Jaringan Lokal

Penulis: Toki • Tanggal: 7 November 2025

1. Pendahuluan

ARP Spoofing adalah teknik serangan pada jaringan lokal yang memanfaatkan kelemahan Address Resolution Protocol (ARP) untuk memposisikan penyerang sebagai perantara antara korban dan gateway. Serangan ini memungkinkan penyerang melakukan pemantauan, modifikasi, atau pencurian data tanpa disadari oleh korban.

2. Tujuan Praktikum

  • Memahami konsep ARP Spoofing pada jaringan lokal
  • Mempelajari mekanisme Man-in-the-Middle (MITM)
  • Menguji penggunaan tools Ettercap dan arpspoof
  • Menganalisis alur lalu lintas data korban

3. Lingkungan Pengujian

  • Attacker: Kali Linux
  • Korban: Host di jaringan lokal
  • Gateway: Router jaringan
  • Interface: wlan0 / eth0

4. Metodologi ARP Spoofing

Penyerang mengirimkan ARP reply palsu ke korban dan gateway, sehingga keduanya mengasosiasikan MAC address penyerang sebagai pihak lawan komunikasi. Akibatnya, seluruh lalu lintas akan melewati mesin penyerang.

5. Implementasi Menggunakan Ettercap

5.1 Instalasi Ettercap

sudo apt update
sudo apt install ettercap-text-only

Atau versi GUI:

sudo apt install ettercap-graphical

5.2 Menjalankan Ettercap (GUI)

sudo ettercap -G

5.3 Langkah Konfigurasi

  1. Pilih menu Sniff → Unified sniffing
  2. Pilih interface (misal: wlan0)
  3. Pilih Hosts → Scan for hosts
  4. Pilih Hosts → Host list
  5. Tandai IP korban sebagai Target 1
  6. Tandai IP gateway sebagai Target 2
  7. Pilih Mitm → ARP poisoning dan centang Sniff remote connections
  8. Pilih Start → Start sniffing

Setelah aktif, seluruh lalu lintas antara korban dan internet akan melewati mesin penyerang.

6. Implementasi Alternatif Menggunakan arpspoof

6.1 Instalasi arpspoof

sudo apt install dsniff

6.2 Menjalankan ARP Spoofing ke Korban

sudo arpspoof -i wlan0 -t 192.168.1.100 192.168.1.1
  • -i wlan0 : Interface jaringan
  • 192.168.1.100 : IP korban
  • 192.168.1.1 : IP gateway

6.3 Menjalankan ARP Spoofing ke Gateway

sudo arpspoof -i wlan0 -t 192.168.1.1 192.168.1.100

Dengan dua perintah ini, penyerang berada di tengah komunikasi korban dan gateway.

7. Hasil Pengujian

  • Lalu lintas korban berhasil dialihkan ke mesin penyerang
  • Koneksi internet korban tetap berjalan tanpa gangguan
  • Data dapat dianalisis menggunakan packet sniffer

8. Analisis Keamanan

ARP Spoofing menunjukkan bahwa protokol ARP tidak memiliki mekanisme autentikasi. Tanpa pengamanan tambahan, jaringan lokal sangat rentan terhadap serangan MITM.

9. Mitigasi

  • Menggunakan static ARP
  • Mengaktifkan DHCP snooping & Dynamic ARP Inspection
  • Menggunakan VPN
  • Mengimplementasikan IDS/IPS

10. Kesimpulan

Praktikum ini membuktikan bahwa ARP Spoofing dapat dilakukan dengan mudah pada jaringan lokal yang tidak dilindungi. Pemahaman terhadap teknik ini penting untuk meningkatkan kesadaran dan penerapan mekanisme keamanan jaringan yang lebih baik.
