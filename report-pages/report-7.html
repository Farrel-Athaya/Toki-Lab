Implementasi Wazuh SIEM & XDR untuk Deteksi Ancaman Terpadu

Penulis: Toki • Tanggal: 10 November 2025

Overview

Wazuh adalah platform keamanan open-source yang menggabungkan kemampuan SIEM (Security Information and Event Management) dan XDR (Extended Detection and Response). Eksperimen ini fokus pada pemantauan beban kerja (workloads) di berbagai lingkungan untuk mendeteksi ancaman, memantau integritas file, dan memastikan kepatuhan regulasi (compliance).

Arsitektur Sistem

Wazuh menggunakan arsitektur berbasis agen yang terdiri dari tiga komponen utama:

  • Wazuh Manager: Server pusat yang menganalisis data dan mengirimkan peringatan (alerts).
  • Wazuh Agent: Sensor ringan yang dipasang pada endpoint (Linux, Windows, Cloud).
  • Elastic Stack: Mengindeks data (Elasticsearch) dan memvisualisasikan data (Kibana).

Instalasi Komponen Utama

1. Konfigurasi Wazuh Manager

Menyiapkan repository dan menginstal manager beserta Filebeat untuk pengiriman log. 

# Tambahkan GPG Key & Repo
curl -s https://packages.wazuh.com/key/GPG-KEY-WAZUH | sudo gpg --import
# Install Package
sudo apt update && sudo apt install wazuh-manager filebeat -y

2. Setup Elasticsearch & Kibana

Elasticsearch digunakan sebagai database storage, sementara Kibana dilengkapi dengan Wazuh Plugin untuk dashboard interaktif. 

# Install Kibana Plugin
sudo -u kibana /usr/share/kibana/bin/kibana-plugin install https://packages.wazuh.com/4.x/ui/kibana/wazuh_kibana-4.5.3_7.17.13-1.zip

Fitur Keamanan Utama

File Integrity Monitoring (FIM)

Mendeteksi setiap perubahan (create/modify/delete) pada direktori kritis secara real-time. 

<syscheck>
  <directories check_all="yes">/etc,/usr/bin,/var/www</directories>
</syscheck>

Vulnerability Detection

Memindai endpoint untuk menemukan software yang belum di-patch berdasarkan basis data CVE terbaru. 

<vulnerability-detector>
  <enabled>yes</enabled>
  <interval>5m</interval>
</vulnerability-detector>

Integrasi Pihak Ketiga

Wazuh dapat mengirimkan alert otomatis ke platform komunikasi atau memverifikasi file berbahaya via VirusTotal.


Contoh Integrasi Slack:

<integration>
  <name>slack</name>
  <hook_url>https://hooks.slack.com/services/YOUR/WEBHOOK/URL</hook_url>
  <level>10</level>
</integration>

Kesimpulan & Troubleshooting

Selama eksperimen, ditemukan bahwa sinkronisasi versi antara Wazuh Manager dan Elastic Stack sangatlah kritis. Jika Kibana tidak dapat terhubung, pastikan status cluster Elasticsearch sehat: 

curl -X GET "localhost:9200/_cluster/health?pretty"
